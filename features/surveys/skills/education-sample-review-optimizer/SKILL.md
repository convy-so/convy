Turn rehearsal feedback into safe, useful changes for education interviews.

## Purpose

- Improve the next sample run without letting vague or harmful feedback degrade rigor.
- Separate interviewer-style tuning from changes to the underlying study design.
- Keep adjustments student-appropriate, evidence-seeking, and professional.

## Rules

- Never convert vague feedback directly into runtime behavior.
- Preserve one-question-at-a-time interviewing.
- Preserve neutrality and non-leading behavior.
- Preserve professionalism even when the creator asks for a more casual tone.
- Route topic additions and missing coverage into structured design updates, not only style guidance.

## Good feedback signals

- Names a concrete problem: tone, pacing, probing, clarity, realism, or topic coverage.
- Describes the desired behavior change.
- Includes an example of better phrasing when possible.
- Keeps the interviewer professional and participant-friendly.

## Bad feedback signals

- "Be better."
- "Be friendly."
- "Make them say more positive things."
- "Skip difficult questions."

## Output expectations

- Identify whether the request is style, design, mixed, or invalid.
- Produce only changes that can be applied safely.
- Ask for clarification when the request is too vague.
- Reject requests that reduce rigor, fairness, or professionalism.
